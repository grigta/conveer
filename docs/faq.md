# FAQ (Frequently Asked Questions)

## Общие вопросы

### Что такое Conveer?

Conveer — это автоматизированная система для создания и прогрева аккаунтов в социальных сетях (VK, Telegram, Mail.ru, Max). Система позволяет:

- Автоматически регистрировать аккаунты
- Прогревать их для повышения траста
- Управлять прокси
- Мониторить статистику через Grafana

### Какие платформы поддерживаются?

- **VK (ВКонтакте)** — полная поддержка
- **Telegram** — полная поддержка
- **Mail.ru** — полная поддержка
- **Max (ok.ru)** — полная поддержка

### Сколько аккаунтов можно создавать?

Ограничения зависят от:
- Количества прокси
- Баланса SMS-провайдера
- Rate limits платформ

Рекомендуемые лимиты:
- До 50 аккаунтов в день на одну платформу
- Интервал между регистрациями: 5-15 минут

## Установка и настройка

### Какие минимальные требования для запуска?

**Development:**
- 2 CPU cores
- 4 GB RAM
- 20 GB disk
- Docker & Docker Compose

**Production:**
- 8+ CPU cores
- 16+ GB RAM
- 100+ GB SSD
- Kubernetes (опционально)

### Как обновить систему?

```bash
# Остановить сервисы
docker-compose down

# Получить обновления
git pull origin main

# Пересобрать и запустить
docker-compose build
docker-compose up -d
```

### Как сделать бэкап данных?

```bash
# MongoDB
docker exec conveer-mongodb mongodump --out /backup
docker cp conveer-mongodb:/backup ./backup

# Redis (если persistence включен)
docker cp conveer-redis:/data/dump.rdb ./backup/
```

## Прокси

### Какие провайдеры прокси поддерживаются?

Поддерживаются любые HTTP API провайдеры. Протестированы:
- Bright Data
- Smartproxy
- Oxylabs
- ProxyLine
- MobileProxy.space

### Какой тип прокси лучше использовать?

| Тип | Плюсы | Минусы | Рекомендация |
|-----|-------|--------|--------------|
| Residential | Низкий fraud score | Дорого | Для регистрации |
| Mobile | Очень низкий fraud score | Очень дорого | Для проблемных случаев |
| Datacenter | Дёшево | Высокий fraud score | Не рекомендуется |
| ISP | Баланс цена/качество | Средний fraud score | Для прогрева |

### Почему прокси помечается как banned?

Причины:
1. Высокий fraud score (>70)
2. VPN/Proxy detection
3. Недоступность (timeout)
4. Блокировка на целевой платформе

Решение: проверьте логи health checker и качество провайдера.

### Как часто происходит ротация прокси?

По умолчанию: за 24 часа до истечения срока действия прокси.
Настраивается через `PROXY_ROTATION_BEFORE_EXPIRY`.

## SMS Активация

### Какие SMS-провайдеры поддерживаются?

- SMS-Activate (основной)
- 5sim.net
- SMSPVA
- GetSMS

### Почему не приходит SMS код?

Возможные причины:
1. Номер уже использовался для этого сервиса
2. Провайдер не поддерживает данный сервис/страну
3. Таймаут ожидания (по умолчанию 2 минуты)
4. Проблемы на стороне провайдера

Решения:
1. Запросить новый номер
2. Сменить страну
3. Попробовать другого провайдера

### Как минимизировать расходы на SMS?

1. Использовать retry mechanism (до 4 попыток)
2. Отменять неудачные активации своевременно
3. Выбирать более дешёвые страны (если платформа позволяет)
4. Мониторить success rate и менять провайдеров

## Прогрев аккаунтов

### Что такое прогрев и зачем он нужен?

Прогрев — это имитация естественной активности нового аккаунта:
- Просмотр профилей
- Лайки
- Подписки
- Комментарии

Это повышает "траст" аккаунта и снижает риск бана.

### Какие сценарии прогрева доступны?

| Сценарий | Длительность | Интенсивность | Для чего |
|----------|--------------|---------------|----------|
| basic | 7 дней | Низкая | Минимальный прогрев |
| advanced | 14 дней | Средняя | Стандартный прогрев |
| custom | Настраивается | Любая | Специфические требования |

### Как создать кастомный сценарий?

```json
POST /api/v1/warming/scenarios
{
  "name": "my_scenario",
  "platform": "vk",
  "daily_actions": [
    {
      "day": 1,
      "actions": [
        {"type": "view_profile", "count": 5},
        {"type": "like_post", "count": 3}
      ]
    },
    {
      "day": 2,
      "actions": [
        {"type": "view_profile", "count": 10},
        {"type": "like_post", "count": 5},
        {"type": "subscribe", "count": 2}
      ]
    }
  ]
}
```

### Аккаунт забанили во время прогрева. Что делать?

1. Проверить качество прокси
2. Уменьшить интенсивность сценария
3. Добавить больше случайности в задержки
4. Использовать сценарий с более длительным периодом

## Мониторинг

### Как посмотреть статистику?

- **Grafana**: http://localhost:3000 (admin/admin)
- **Prometheus**: http://localhost:9090
- **RabbitMQ**: http://localhost:15672 (guest/guest)

### Какие метрики доступны?

Основные метрики:
- `accounts_total` — общее количество аккаунтов
- `accounts_by_status` — аккаунты по статусам
- `proxies_available` — доступные прокси
- `warming_tasks_active` — активные задачи прогрева
- `http_request_duration_seconds` — latency API

### Как настроить алерты?

Алерты настраиваются в `monitoring/prometheus/alerts.yml`:

```yaml
groups:
  - name: conveer
    rules:
      - alert: LowProxyPool
        expr: proxies_available < 10
        for: 5m
        labels:
          severity: warning
```

## Безопасность

### Как защищены пароли аккаунтов?

- Шифрование AES-256 GCM
- Ключ шифрования хранится в переменной окружения
- Расшифровка только при необходимости

### Безопасно ли хранить токены в .env?

Для production рекомендуется:
- Docker Secrets
- Kubernetes Secrets
- HashiCorp Vault
- AWS Secrets Manager

### Как ограничить доступ к API?

1. Bearer токен (JWT)
2. Rate limiting
3. IP whitelist (настраивается в nginx/ingress)

## Troubleshooting

### Сервис не запускается

```bash
# Проверить логи
docker-compose logs service-name

# Проверить переменные окружения
docker-compose config

# Проверить зависимости
make health
```

### API возвращает 500 ошибку

1. Проверить логи сервиса
2. Проверить подключение к MongoDB/Redis/RabbitMQ
3. Проверить метрики на наличие аномалий

### Действия прогрева не выполняются

1. Проверить статус задачи
2. Проверить очередь RabbitMQ
3. Проверить логи warming-service
4. Проверить, не забанен ли аккаунт

## Разработка

### Как добавить поддержку новой платформы?

1. Создать сервис в `services/newplatform-service/`
2. Реализовать интерфейс `PlatformExecutor`
3. Добавить gRPC методы
4. Обновить warming-service для поддержки новой платформы
5. Создать сценарии прогрева

### Как запустить тесты локально?

```bash
# Все тесты
make test

# С покрытием
make test-coverage

# Только unit тесты
make test-unit
```

### Как контрибьютить в проект?

1. Fork репозитория
2. Создать feature branch
3. Написать код и тесты
4. Убедиться, что линтер проходит: `make lint`
5. Создать Pull Request
